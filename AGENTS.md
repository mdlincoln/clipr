# clipr — Agent Instructions

## Repo type

R package (`clipr`): read/write system clipboard on macOS
(pbpaste/pbcopy), Windows (readClipboard/writeClipboard), X11
(xclip/xsel), and Wayland (wl-copy/wl-paste).

## Key commands

- `R -e "devtools::check()"` — full R CMD check with vignettes
- `R -e "testthat::test_local()"` — run all tests
- `R -e "roxygen2::roxygenise()"` — regenerate man/ from roxygen
  comments (NAMESPACE is auto-generated)
- `R -e "pkgdown::build_site()"` — build docs

## Non-interactive mode: CLIPR_ALLOW

All public functions (`read_clip`, `write_clip`, etc.) **refuse to run
in non-interactive sessions** unless `CLIPR_ALLOW=TRUE` is set as an env
var or passed as `allow_non_interactive = TRUE`. Tests use this gate. Do
not attempt to run clipr on CRAN — xclip/xsel fork background processes
that cause check failures
([\#38](https://github.com/mdlincoln/clipr/issues/38)).

## Test structure

- testthat suite in `tests/testthat/`, driven by `tests/testthat.R`
  (`test_check("clipr")`)
- `setup.R` sets up `is_clipr_available` / `skip_msg` — every functional
  test calls `skip_if_not(is_clipr_available, skip_msg)`
- CI runs R CMD check across macOS, Windows, and Ubuntu (xclip, xsel,
  wayland, none, nodisplay) with XVFB on Linux

## File ownership

| Area | Files |
|----|----|
| Public API | `R/clipboard.R` — exports: `read_clip`, `write_clip`, `clear_clip`, `write_last_clip`, `read_clip_tbl`, `clipr_available`, `dr_clipr` |
| Platform backends | `R/osx_clipboard.R`, `R/win_clipboard.R`, `R/linux_clipboard.R` |
| Rendering logic | `R/flat_str.R` — `render_object()`, `table_str()`, `flat_str()` |
| Utilities | `R/utils.R` — `sys_type()`, `clipr_available_handler()`, messages, `.onAttach` |

## Conventions

- README.md is **generated from** `README.Rmd` — edit the Rmd source
  only
- NAMESPACE and man/ are auto-generated by roxygen2 (v7.3.0) — never
  hand-edit
- Encoding: UTF-8, Language: en-US
- Line endings are platform-aware in tests: Windows uses `\r\n`,
  Unix/macOS use `\n`

- `pkgdown.yaml` — docs deployment
